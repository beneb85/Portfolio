Vertical52

The Vertical52 redesign transforms a complex satellite image platform into an intuitive investigative workspace for journalists. It enables users to explore imagery, manage investigations, and analyze spatial data within a unified experience. By centralizing workflows, improving navigation, and enhancing the interface and interactions, the platform shifts from a fragmented toolset to a cohesive product—supporting faster, clearer, and more effective investigations.

Client Vertical52
Timeline 2025
Role UX / UI Designer
Impact International Reach

The Challenge

At its core, the challenge was to design for complex, data-heavy workflows while maintaining clarity and usability. Users needed to move seamlessly from data access to insight generation.

Key friction points included:

  • Overwhelming information density across market dashboards
  • No clear hierarchy between primary data and supporting context
  • Inconsistent patterns that slowed expert users and confused newcomers
  • A mobile experience that was an afterthought, not a first-class citizen

The platform needed to serve two very different users simultaneously: analysts who live in the data, and executives who need a 30-second read of the market.

The Solution

A redesigned information architecture that separates signal from noise — giving each user type an entry point that respects their context without hiding depth.

Key design decisions:

  • Progressive disclosure: summary cards surface key metrics first, detail layers reveal on demand
  • A unified component system that scales across six dashboard views without repetition
  • Contextual filters that persist across sessions, reducing repetitive setup time
  • Responsive-first layout that adapts data tables to mobile without losing functionality

The result is a platform that handles real estate complexity with the same directness a professional expects from their tools — fast, trustworthy, and built for daily use.

↓ 40% Reduction in time-to-insight for analyst workflows after redesign
6 Dashboard modules unified under a single component system
3× Increase in mobile session length compared to previous version
Information Architecture

Clarity at every scale

The dashboard was rebuilt around a clear hierarchy: summary → trend → detail. Each level rewards attention without demanding it. Colour, weight, and space all carry meaning — nothing is decorative.

Design System

One system, six surfaces

Rather than designing six dashboards independently, every module was built from a shared set of primitives. Tokens for spacing, colour, and typography ensured visual consistency across contexts — and made handoff to engineering straightforward.

Responsive Design

Data tables that actually work on mobile

Dense tables were redesigned using a card-first approach on small screens, with horizontal scroll locked to data-critical columns. Professionals checking the market between meetings shouldn't have to pinch and zoom to find the number that matters.
